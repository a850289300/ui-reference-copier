# UI Reference Copier

UI Reference Copier 是一个免费的 Chrome MV3 插件，用来从参考页面、原型页面或设计预览页面采集浏览器真实渲染后的元素信息，并一键复制成适合 Codex、Claude Code、Cursor 等 AI 编程工具使用的提示词。

它解决的问题很具体：模型只看截图或 HTML 时，经常还原不准。这个插件会采集真实的尺寸、字体、颜色、间距、布局、组件变量和关键子元素样式，帮助 AI 更接近 1:1 还原页面。

## 适合谁

- 后端工程师：不需要打开 DevTools，也能把参考 UI 信息复制给 AI。
- 前端工程师：快速采集 computed style 和跨页面差异，减少人工对 CSS。
- 设计还原场景：从原型页、参考页、MasterGo 预览页采集元素，用于实现或修复页面。

## 功能亮点

- 点击页面元素，复制 AI 还原提示词。
- 默认输出 token 友好的摘要，不塞完整 JSON。
- 可选复制完整 computed CSS 或结构化 JSON。
- 支持多选元素。
- 支持拖拽面板。
- 支持选择父级元素，方便选中卡片、容器、网格区域。
- 面板会用人话显示当前选择，例如卡片、按钮、进度条、网格区域；技术 selector 仍保留为辅助信息。
- 支持子元素采样：精简 20、标准 50、详细 100。
- 单个采集默认保留 selector、DOM path 和 HTML，方便模型理解当前项目元素；如果采集的是外部参考页，可开启「外部参考页模式」提醒模型不要照搬参考类名。
- 支持跨页面保存参考元素并对比当前实现。
- 支持多组元素对比：可以分别保存访问量卡片、销售额卡片、订单量卡片等参考组，再逐组匹配当前实现。
- 差异报告会先给关键差异摘要，再附详细信息，可一键复制给 AI 生成修复代码。
- 多组对比默认复制精简差异，避免 2000 行长提示词；需要完整细节时可单独复制详细差异。
- 图标细节采集默认关闭，可在需要对比 svg、iconfont、img 图标时手动开启。
- 对 Naive UI 等组件库会采集 `--n-*` 组件变量。
- 复制成功或失败会显示右上角提示，不遮挡操作。

## 安装

1. 打开 `chrome://extensions/`
2. 开启右上角「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择本项目目录

## 使用

1. 打开参考页面、原型页面或 MasterGo 预览页面。
2. 点击浏览器右上角 `UI Reference Copier` 插件图标。
3. 鼠标悬停页面元素，点击要复刻的元素；按住 Cmd / Ctrl / Shift 点击可追加或取消多选。
4. 右侧面板会显示视觉摘要；可在「采集 / 单组对比 / 多组对比」之间切换，插件会记住上次使用的 tab。
5. 点击「复制给 AI」「复制完整样式」或「复制 JSON」。

如果选中了太小的文字或图标，可以点击「选择父级」切换到更大的容器；采集、单组对比、多组对比页都支持这个按钮。

默认「复制给 AI」不会携带完整 JSON，避免浪费 token；需要结构化数据时单独点「复制 JSON」。
如果当前页面是外部参考站点，可以开启「外部参考页模式」。开启后，单个复制会提醒 AI：参考页的 class、id、selector 只用于理解结构，不要直接照搬到当前项目。

## 跨页面对比

1. 在参考页选中一个或多个元素，点击「设为参考」。
2. 切换到当前实现页，选中对应元素。
3. 点击「对比参考」生成差异报告。
4. 点击「复制差异给 AI」让模型按差异修复当前项目。

快捷键：插件面板开启后，选中元素时可按 `Cmd + S` / `Ctrl + S` 快速「设为参考」。

多选时按选择顺序对比：参考第 1 个对当前第 1 个，参考第 2 个对当前第 2 个。
参考状态通过 Chrome 本地存储同步；一个页面清除参考后，其他已打开页面的插件面板会自动更新。
对比提示词会弱化参考页 selector，但保留当前实现页 selector，方便 AI 明确应该修改当前项目里的哪个元素。

## 多组元素对比

适合一个页面里有多个需要分别校准的区域，比如指标卡片、图表卡片、表格和操作栏。

1. 在参考页选中第一组元素，点击「保存新参考组」。
2. 继续选择其他区域，重复保存多个参考组。
3. 切换到当前实现页，在下拉框里选择要匹配的组。
4. 选中当前页面对应元素，点击「匹配当前组」。
5. 每组都匹配后，点击「对比全部组」。
6. 点击「复制精简差异」，把按组整理好的修复提示词交给 Codex / Claude Code。

多组提示词会额外包含整体布局关系和每组范围快照，帮助 AI 判断参考到底是外层卡片、内部图标文字，还是整组容器。没有匹配当前实现的组会在提示词里标出来，避免漏修。
点击「对比全部组」时会重新计算最新差异；如果参考像外层卡片但当前像内部元素，提示词会标出范围风险，提醒先重新选择正确层级。
如果需要全部逐元素差异，可以点击「复制详细差异」。
匹配当前组成功后，插件会自动切到下一个还没匹配的参考组，减少反复操作下拉框。

快捷键：切到「多组对比」后，选中元素时可按 `Cmd + S` / `Ctrl + S` 保存新参考组，按 `Cmd + D` / `Ctrl + D` 匹配当前组。

## 采集内容

- 页面 URL、标题、viewport、DPR
- 单个或多个元素的 tag、文本、selector、DOM path
- 元素位置和尺寸
- 字体、字号、字重、行高、颜色、背景、padding、margin、border、圆角、阴影
- display、flex/grid、gap、align、justify、position
- 父级布局上下文
- 容器内关键子元素采样，用于发现标题、按钮、图标、图片等内部差异
- 子元素采样可选：精简 20、标准 50、详细 100
- 简短 outerHTML
- 面向 Codex / Claude Code 的中文 prompt
- 可选完整 computed CSS prompt
- 可选图标摘要：svg viewBox、path 数量、use href、img src、iconfont class 等
- 跨页面差异报告和修复 prompt
- 多组差异报告和按组修复 prompt

## 限制

- 目前没有裁剪元素截图。
- 不会自动改代码，只负责复制高质量参考上下文。
- 子元素对比是关键节点采样，不是完整 DOM 树像素级 diff；可在面板里调整采样深度。
- 子元素匹配不依赖类名完全一致，会综合文本、tag、role/aria、相对位置、尺寸和选择顺序打分匹配。
- iframe 内部元素需要在 iframe 可访问时才可能采集。
- 某些站点的扩展限制或 CSP 可能影响注入。

## 开发

需要 Node.js 18+。

```bash
npm test
npm run check
npm run pack
```

`npm run pack` 会在上级目录生成 `ui-reference-copier.zip`。
