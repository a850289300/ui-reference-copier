# UI Reference Copier

UI Reference Copier 是一个免费的 Chrome MV3 插件，用来从参考页面、原型页面或设计预览页面采集浏览器真实渲染后的元素信息，并一键复制成适合 Codex、Claude Code、Cursor 等 AI 编程工具使用的提示词。

它解决的问题很具体：模型只看截图或 HTML 时，经常还原不准。这个插件会采集真实的尺寸、字体、颜色、间距、布局、组件变量和关键子元素样式，帮助 AI 更接近 1:1 还原页面。

## 适合谁

- 后端工程师：不需要打开 DevTools，也能把参考 UI 信息复制给 AI。
- 前端工程师：快速采集 computed style 和跨页面差异，减少人工对 CSS。
- 设计还原场景：从原型页、参考页、MasterGo 预览页采集元素，用于实现或修复页面。

## 功能亮点

- 点击页面元素，复制 AI 还原提示词。
- 默认输出 token 友好的摘要，不塞完整 JSON。
- 可选复制完整 computed CSS 或结构化 JSON。
- 支持多选元素。
- 支持拖拽面板。
- 支持选择父级元素，方便选中卡片、容器、网格区域。
- 面板会用人话显示当前选择，例如卡片、按钮、进度条、网格区域；技术 selector 仍保留为辅助信息。
- 支持子元素采样：单个采集可选不采集子元素，或使用精简 20、标准 50、详细 100。
- 单个采集默认保留 selector、DOM path 和 HTML，方便模型理解当前项目元素；如果采集的是外部参考页，可开启「外部参考页模式」提醒模型不要照搬参考类名。
- 支持跨页面保存参考元素并对比当前实现。
- 支持多组元素对比：可以分别保存访问量卡片、销售额卡片、订单量卡片等参考组，再逐组匹配当前实现。
- 支持独立「结构对比」：当两个页面结构差异太大时，先判断是否选错层级或缺少布局区域，再让 AI 修 DOM / 组件结构。
- 结构对比默认只复制关键差异点和修改建议，不输出完整 DOM 明细；复杂场景可单独复制详细结构数据。
- 针对菜单/导航结构会额外输出菜单语义摘要，提醒 AI 保留当前项目组件库，只同步菜单项、层级、顺序、图标位置、选中态和展开态。
- 差异报告会先给关键差异摘要，再附详细信息，可一键复制给 AI 生成修复代码。
- 多组对比默认复制精简差异，避免 2000 行长提示词；需要完整细节时可单独复制详细差异。
- 图标细节采集默认关闭，可在需要对比 svg、iconfont、img 图标时手动开启；会采集当前选中元素本身和内部子图标。
- 支持独立「取色」：像吸管一样点击页面颜色点，只复制吸到的颜色提示词或颜色值，不要求 AI 改布局和结构。
- 对 Naive UI 等组件库会采集 `--n-*` 组件变量。
- 复制成功或失败会显示右上角提示，不遮挡操作。

## 安装

1. 打开 `chrome://extensions/`
2. 开启右上角「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择本项目目录

## 使用

1. 打开参考页面、原型页面或 MasterGo 预览页面。
2. 点击浏览器右上角 `UI Reference Copier` 插件图标。
3. 鼠标悬停页面元素，点击要复刻的元素；按住 Cmd / Ctrl / Shift 点击可追加或取消多选。
4. 右侧面板会显示视觉摘要；可在「采集 / 单组对比 / 多组对比 / 结构对比 / 取色」之间切换，插件会记住上次使用的 tab。
5. 点击「复制给 AI」「复制完整样式」或「复制 JSON」。

如果选中了太小的文字或图标，可以点击「选择父级」切换到更大的容器；采集、单组对比、多组对比页都支持这个按钮。

默认「复制给 AI」不会携带完整 JSON，避免浪费 token；需要结构化数据时单独点「复制 JSON」。
如果当前页面是外部参考站点，可以开启「外部参考页模式」。开启后，单个复制会提醒 AI：参考页的 class、id、selector 只用于理解结构，不要直接照搬到当前项目。

## 跨页面对比

1. 在参考页选中一个或多个元素，点击「设为参考」。
2. 切换到当前实现页，选中对应元素。
3. 点击「对比参考」生成差异报告。
4. 点击「复制差异给 AI」让模型按差异修复当前项目。

快捷键：插件面板开启后，选中元素时可按 `Cmd + S` / `Ctrl + S` 快速「设为参考」。

多选时按选择顺序对比：参考第 1 个对当前第 1 个，参考第 2 个对当前第 2 个。
参考状态通过 Chrome 本地存储同步；一个页面清除参考后，其他已打开页面的插件面板会自动更新。
对比提示词会弱化参考页 selector，但保留当前实现页 selector，方便 AI 明确应该修改当前项目里的哪个元素。
如果两个元素的结构相似度偏低，差异提示词会自动加入结构风险提示，建议先用「结构对比」修布局，再继续处理样式。
「样式对比范围」可以选择只对比当前元素，或包含采样子元素。跨不同 UI 组件库对齐时，建议先用「只对比当前元素」确认根容器、尺寸、背景、字体和结构风险；如果根容器已经对齐，再切换到「包含子元素」排查内部标题、按钮、图标和局部样式。

## 取色

如果只是想把一个页面的颜色吸出来，再让 AI 设置到当前项目的对应元素，不需要走完整样式对比。

1. 切到「取色」。
2. 点击「开始吸色」。
3. 使用 Chrome 原生吸管点击要吸取的屏幕颜色点。可以重复点击，保存多个颜色。
4. 切到当前实现页，点击「选择目标元素」，再点页面里要改颜色的元素。
5. 在「应用到」里选择自动判断、背景色、文字色、边框色、图标色、阴影色或主题变量。
6. 点击「复制颜色修改提示词」，交给 Codex / Claude Code 只同步目标元素颜色。
7. 如果只想手动查看，可点击「复制颜色值」。
8. 如果当前组件库使用主题变量，也可以先用普通方式选中元素，再点击「复制元素颜色变量」。

默认吸色不会圈选整个元素，也不会采集所有子元素。Chrome 支持原生 `EyeDropper` 时会直接读取屏幕像素颜色；如果当前浏览器不支持，才回退到页面样式吸色，尽量读取鼠标点击位置命中的背景色、文本色、边框色、SVG fill/stroke 或渐变背景。

吸到的颜色、目标元素和「应用到」选择会通过 Chrome 本地存储同步，所以可以先在参考页吸色，再切到目标页设置目标元素。取色提示词会明确告诉 AI：吸色不是默认改文本色，而是根据目标元素用途或你的手动选择，应用到 `background-color`、`color`、`border-color`、SVG `fill/stroke`、阴影颜色或主题变量等任意颜色属性；不改布局、尺寸、字体、DOM 结构和交互逻辑。颜色来源 selector 只用于理解颜色来自哪里，不要照搬到当前项目。

「复制元素颜色变量」属于高级用法，会从当前选中元素里筛选颜色相关 CSS 变量，例如名称里包含 `color`、`fill`、`stroke`、`border`、`text`、`bg`、`background`、`rail`、`shadow`、`mask` 的变量。图标细节采集默认关闭，如果要分析 iconfont、mask 图标变量，需要先开启「元素颜色模式采集图标细节」，再重新选择元素。

## 多组元素对比

适合一个页面里有多个需要分别校准的区域，比如指标卡片、图表卡片、表格和操作栏。

1. 在参考页选中第一组元素，点击「保存新参考组」。
2. 继续选择其他区域，重复保存多个参考组。
3. 切换到当前实现页，在下拉框里选择要匹配的组。
4. 选中当前页面对应元素，点击「匹配当前组」。
5. 每组都匹配后，点击「对比全部组」。
6. 点击「复制精简差异」，把按组整理好的修复提示词交给 Codex / Claude Code。

多组提示词会额外包含整体布局关系和每组范围快照，帮助 AI 判断参考到底是外层卡片、内部图标文字，还是整组容器。没有匹配当前实现的组会在提示词里标出来，避免漏修。
点击「对比全部组」时会重新计算最新差异；如果参考像外层卡片但当前像内部元素，提示词会标出范围风险，提醒先重新选择正确层级。
如果需要全部逐元素差异，可以点击「复制详细差异」。
多组对比同样支持「样式对比范围」。如果参考和当前来自不同 UI 组件库，先只对比当前元素可以减少内部 DOM 差异噪音。
匹配当前组成功后，插件会自动切到下一个还没匹配的参考组，减少反复操作下拉框。

快捷键：切到「多组对比」后，选中元素时可按 `Cmd + S` / `Ctrl + S` 保存新参考组，按 `Cmd + D` / `Ctrl + D` 匹配当前组。

## 结构对比

当样式差异很多、AI 怎么改都差一点时，常见原因是两个页面选中的不是同一层级：参考选的是外层卡片，当前选成了内部文字；或者当前实现缺少 header、footer、图表、表格、操作区等结构。

1. 切到「结构对比」。
2. 在参考页选中正确范围，点击「设为结构参考」。
3. 切到当前实现页，选中对应范围，点击「设为当前结构」。
4. 点击「对比结构」。
5. 点击「复制结构差异点」，默认只把关键差异和建议复制给 AI。
6. 如果页面很复杂、需要排查完整统计，再点击「复制详细结构数据」。

结构对比会看根节点、display、子元素数量、tag/角色分布、文本节点、媒体节点和是否像可见卡片。它不关注颜色、字体这些细节，目标是先判断“两个东西是不是同一个层级的东西”。

结构对比支持单独选择采样深度：精简 24 个、中等 80 个、全量已采集子元素。普通卡片或布局建议用中等；长菜单、侧边栏导航、复杂导航树建议切到全量，并重新保存结构参考和当前结构后再对比。

如果对比对象像菜单、侧边栏导航或导航树，结构对比会额外输出「组件语义差异」。这类提示不会要求模型照搬参考页的 `div`、`ul`、`li` 或具体 UI 库 DOM，而是要求优先保留当前项目已有菜单组件、路由配置或菜单数据源，只同步菜单项文本、层级、顺序、分组、图标位置、选中态和展开态。

在普通样式差异里，如果插件识别到当前对比对象是菜单/导航，也会进入「菜单组件对齐模式」。这个模式不会展开普通子元素逐项样式 diff，也会忽略参考组件库专属变量，例如 `--n-*`，避免模型被不同 UI 库内部 DOM 和变量名带偏。提示词会改为强调菜单树、菜单根容器、一级菜单行、子菜单行、图标、文字、箭头、active/opened/hover 状态等真实还原目标。

结构对比分三档提示：

- 蓝色：结构基本一致，可以继续修样式。提示词仍会列出少量局部结构差异，例如标题、进度条、图表或统计项是否缺失。
- 橙色：结构可能不一致，建议先确认是否选中了同一层级；如果层级正确，再补齐关键内部结构。
- 红色：结构明显不一致，先不要修样式；优先重新选择外层容器，或先重建 DOM / 组件布局。

默认结构提示词不会列完整 DOM 树，也不会把 90 个对 75 个节点全部塞给模型。它只输出真正可能影响还原的差异点，例如：

- 参考元素和当前元素的短身份摘要：名称、位置尺寸、短文本摘要、参考定位 selector 和当前目标 selector；参考 selector 只用于识别范围，不要照搬到当前项目。
- 文本摘要会优先保留业务关键词，例如访问量、销售额、订单量、成交额，并过滤动态金额、百分比和大段随机数字。
- 参考有 `progressbar` 区域，当前没有匹配到。
- 参考有 heading 角色，当前被普通文本替代。
- 当前文本节点明显更多，可能是结构被拍平。
- 参考像外层卡片，但当前像内部文字或透明容器。

详细结构数据会额外包含 tag/role 分布、逐元素结构数据、selector、DOM path、父级布局和完整统计，用于复杂页面排查，不建议默认复制给 AI 硬改。

## 采集内容

- 页面 URL、标题、viewport、DPR
- 单个或多个元素的 tag、文本、selector、DOM path
- 元素位置和尺寸
- 字体、字号、字重、行高、颜色、背景、padding、margin、border、圆角、阴影
- display、flex/grid、gap、align、justify、position
- 父级布局上下文
- 容器内关键子元素采样，用于发现标题、按钮、图标、图片等内部差异
- 单个采集可关闭子元素采样，用来减少 token；做 1:1 细节还原或差异对比时建议保留采样。
- 子元素采样可选：不采集、精简 20、标准 50、详细 100
- 简短 outerHTML
- 面向 Codex / Claude Code 的中文 prompt
- 可选完整 computed CSS prompt
- 可选图标摘要：svg viewBox、path 数量、use href、img src、iconfont class、font family、mask/background image 等
- 取色摘要：颜色提示词、纯颜色值、颜色 CSS 变量
- 跨页面差异报告和修复 prompt
- 多组差异报告和按组修复 prompt
- 结构差异点 prompt 和详细结构数据 prompt

## 限制

- 目前没有裁剪元素截图。
- 不会自动改代码，只负责复制高质量参考上下文。
- 子元素对比是关键节点采样，不是完整 DOM 树像素级 diff；可在面板里调整采样深度。
- 子元素匹配不依赖类名完全一致，会综合文本、tag、role/aria、相对位置、尺寸和选择顺序打分匹配。
- iframe 内部元素需要在 iframe 可访问时才可能采集。
- 某些站点的扩展限制或 CSP 可能影响注入。

## 开发

需要 Node.js 18+。

```bash
npm test
npm run check
npm run pack
```

`npm run pack` 会在上级目录生成 `ui-reference-copier.zip`。
