# Improve Menu Structure Prompts

## Why

结构对比当前主要输出根节点、tag、role 和子元素数量差异。遇到菜单组件时，不同 UI 库的真实 DOM 往往差异很大，例如一个库使用 `div`，另一个库使用 `ul/li/span`。这会让提示词过度强调 DOM tag 差异，模型容易误以为需要照搬参考页面 DOM，而不是继续使用当前项目已有菜单组件。

用户的真实目标是：在菜单、侧边栏导航、导航树等场景中，保留当前项目技术栈和组件库，只同步参考页面的菜单语义结构，包括菜单项文本、层级、顺序、分组、图标线索、选中态和展开状态。

## What Changes

- 为结构对比增加通用菜单/导航语义识别。
- 在结构提示词中新增菜单语义差异段落。
- 对菜单场景弱化默认提示词里的 DOM tag 数量差异，避免误导模型照搬参考 DOM。
- 输出参考菜单结构、当前菜单结构、缺失项、多余项和层级差异摘要。
- 明确提示模型优先使用当前项目已有菜单组件、路由配置或菜单数据源，而不是复制参考页面 class、selector 或 UI 库 DOM。
- 增加测试覆盖菜单语义提示。
- 新增项目实现文档，记录本次功能背景和实现策略。

## Out of Scope

- 不做特定 UI 库适配，不写死 Naive UI、Element Plus、Ant Design、Arco 等库之间的转换规则。
- 不自动修改目标项目代码。
- 不做完整 DOM tree diff 或像素级菜单还原。
- 不识别业务路由文件，只基于采集到的浏览器渲染结构生成提示词。

## Success Criteria

- 当结构对比对象像菜单或导航时，提示词能告诉模型这是菜单语义差异，不应直接按 DOM tag 数量重写。
- 默认提示词包含参考菜单树和当前菜单树的关键差异。
- 当前项目 selector 仍然保留，用于告诉模型要修改哪个当前元素。
- 详细版仍保留原有 tag/role/DOM path 统计，方便排查。
- 原有卡片、指标、进度条等结构对比输出不被破坏。
